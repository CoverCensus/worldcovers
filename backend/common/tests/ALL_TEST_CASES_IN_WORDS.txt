================================================================================
COMMON APP TEST SUITE — PLAIN-LANGUAGE DESCRIPTIONS
================================================================================
This document lists every automated test under backend/common/tests/ in words
you can copy for specs, checklists, or documentation. Names match test methods.

--------------------------------------------------------------------------------
test_api_auth.py — Session auth API (/api/v2/)
--------------------------------------------------------------------------------

LoginViewTest
  • test_login_with_username_and_password — POST login with username and
    password returns 200 and user payload includes that username.
  • test_login_with_email_address — POST login with email and password returns
    200 and user payload includes that email.
  • test_login_with_wrong_password_returns_401 — Wrong password returns 401.
  • test_login_missing_credentials_returns_400 — Empty body returns 400.
  • test_login_inactive_user_returns_403 — Inactive user cannot log in; status
    is either 401 or 403 (depends on auth backend behavior).

CurrentUserViewTest
  • test_anonymous_returns_401 — GET current user without session returns 401.
  • test_authenticated_returns_user_payload — Logged-in user gets 200, username
    matches, and response includes a role field.

LogoutViewTest
  • test_logout_after_login_clears_session — After POST logout, GET current user
    returns 401 (session cleared).

LoginRequestViewTest
  • test_creates_inactive_user — POST login-request creates user with email,
    inactive, no usable password (201).
  • test_existing_email_rejected — Duplicate email returns 400.

ForgotPasswordApiViewTest (email backend: in-memory)
  • test_missing_email_returns_400 — No email in body returns 400.
  • test_unknown_email_returns_400 — Email not in database returns 400.
  • test_known_email_sends_message — Valid email returns 200, one outbound
    email, recipient correct, body contains uid= and token= for SPA reset.

ResetPasswordApiViewTest
  • test_resets_password_when_token_valid — Valid uid/token/new password returns
    200 and user password updates in DB.
  • test_rejects_invalid_token — Bad token returns 400.
  • test_rejects_missing_fields — Incomplete body (e.g. missing password fields)
    returns 400.
  • test_rejects_too_short_password — Password too short returns 400.

ChangePasswordApiViewTest
  • test_changes_password_when_inputs_valid — Logged-in user with correct current
    password and valid new password returns 200 and password updates.
  • test_rejects_wrong_current_password — Wrong current password returns 400.
  • test_rejects_weak_password — New password failing strength rules returns 400.
  • test_requires_authentication — Not logged in returns 403.

test_api_contributions.py — Contributions (submit, list, editor actions)
--------------------------------------------------------------------------------

ContributionSubmitTest
  • test_unauthenticated_post_rejected — POST contribution list without auth
    returns 401 or 403.
  • test_state_is_required — Missing state returns 400; error mentions state.
  • test_unknown_state_rejected — Unknown state name returns 400.
  • test_invalid_type_rejected — Invalid contribution type returns 400.
  • test_successful_submission_creates_pending_contribution — Valid submit
    returns 201; contribution is pending, tied to correct collection and
    contributor; a SUBMIT submission transaction exists.
  • test_can_be_saved_as_draft — save_as_draft creates contribution in DRAFT
    status (201).

ContributionListAccessTest
  • test_contributor_sees_only_their_own — Contributor listing returns only rows
    where they are the contributor.
  • test_assigned_editor_sees_contributions — Assigned editor’s list returns at
    least one contribution.

ContributionEditorEditTest
  • test_merges_payload_into_submitted_data — Editor PATCH merges fields into
    submitted_data, preserves untouched keys, records EDITOR_EDIT transaction.
  • test_non_pending_contribution_rejected — Cannot editor-edit non-pending (e.g.
    approved) contribution; returns 400.
  • test_non_dict_payload_rejected — Non-object JSON body returns 400.

ContributionRejectTest
  • test_assigned_editor_can_reject — Editor rejects pending contribution: 200,
    status REJECTED, reviewer and notes set, REJECT transaction logged.
  • test_cannot_reject_non_pending_contribution — Rejecting already-rejected
    returns 400.
  • test_unassigned_editor_cannot_see_contribution — Editor not assigned to
    collection gets 403 or 404 on reject (no existence leak).

ContributionApproveTest
  • test_approve_returns_501_while_apply_pipeline_unimplemented — Approve POST
    returns 501; contribution stays pending (apply not implemented).

test_api_lookups.py — Lookup APIs (regions, post offices, FAQ, collections, etc.)
--------------------------------------------------------------------------------

RegionViewSetTest
  • test_anonymous_can_list_regions — Unauthenticated GET region list returns 200.
  • test_assigned_only_returns_only_users_regions — Logged-in editor with
    assigned_only=true sees only regions tied to assigned collections (e.g.
    Virginia yes, Ohio no).
  • test_assigned_only_for_anonymous_returns_empty_results — Anonymous with
    assigned_only returns 200 and empty list.
  • test_assigned_only_sets_no_store_cache — Response Cache-Control includes
    no-store for assigned_only listing.
  • test_search_by_abbrev — Search parameter finds region by abbreviation (e.g.
    OH → Ohio).

PostOfficeTownOptionsTest
  • test_returns_unique_town_state_pairs — Endpoint returns distinct town/state
    pairs including expected Richmond/Virginia and Columbus/Ohio.

FAQEntryViewSetTest
  • test_public_endpoint_lists_only_active — FAQ list returns only active entries
    (inactive hidden).

CollectionViewSetPermissionsTest
  • test_anyone_authenticated_can_list — Any logged-in user can GET collections.
  • test_anonymous_cannot_list_collections — Anonymous GET returns 403.
  • test_non_admin_cannot_create — Non-admin POST new collection returns 403.
  • test_admin_can_assign_and_unassign_editor — Admin POST assign editor returns
    201 and assignment exists; GET editors lists them; DELETE unassign returns
    204 and assignment removed.
  • test_unassign_missing_assignment_returns_404 — Unassign editor who was never
    assigned returns 404.

LookupCRUDDefaultsTest (Color / Shape serializers)
  • test_create_color_without_user_fks_currently_returns_400 — POST color
    without created_by/modified_by returns 400 with those field errors (serializer
    quirk documented in test).
  • test_create_color_succeeds_when_user_fks_are_supplied — POST with user FKs
    returns 201; audit FKs match request user after create.
  • test_create_shape_succeeds_when_user_fks_are_supplied — POST shape with user
    FKs returns 201.
  • test_anonymous_cannot_create_lookup — Anonymous POST color returns 403.

test_api_markings.py — Markings API and related routes
--------------------------------------------------------------------------------

MarkingListEndpointTest
  • test_list_is_public_readable — Anonymous GET marking list returns 200 with
    expected marking codes sorted.
  • test_list_supports_type_filter — Filter by type (e.g. RATEMARK) returns only
    matching codes.
  • test_list_includes_state_annotation — List rows include state name and
    state abbreviation from region.

MarkingDetailEndpointTest
  • test_retrieve_returns_full_detail — GET detail returns 200 with citations,
    images, and correct id.

MarkingCreateUpdateDeleteTest
  • test_create_requires_authentication — Anonymous POST returns 401 or 403.
  • test_create_emits_record_create_transaction — Editor create returns 201;
    RECORD_CREATE submission transaction exists; marking has one version.
  • test_update_records_change — PATCH updates field; RECORD_UPDATE transaction
    exists.
  • test_delete_emits_record_delete_transaction — DELETE returns 204; marking
    gone; RECORD_DELETE transaction with deleted_marking_id in extra_payload.

MarkingChangelogTest
  • test_returns_events_and_versions — After two PATCH edits, changelog GET
    returns marking id, at least two events, at least two versions.
  • test_changelog_denied_for_unrelated_editor — Editor not assigned to marking’s
    region/collection gets 403 on foreign marking’s changelog.

MarkingDateRangeViewTest
  • test_returns_null_when_no_data — Range endpoint returns null earliest/latest
    year when no date data.
  • test_returns_min_and_max_years — With cover dates seen, returns min/max years
    across aggregated DateSeen data.

DeleteMyMarkingViewTest
  • test_invalid_id_returns_400 — Non-numeric id in URL yields 400 or 404 (not
    success).
  • test_missing_marking_returns_404 — Valid numeric id with no marking returns
    404.
  • test_superuser_can_delete_any — Superuser DELETE delete-mine removes marking
    (204).
  • test_contributor_cannot_delete_others_marking — Contributor cannot delete
    admin’s marking (403); marking remains.

test_audit.py — common.audit helpers (snapshots, diff, transactions, restore)
--------------------------------------------------------------------------------

BuildMarkingSnapshotTest
  • test_empty_for_none — build_marking_snapshot(None) is empty dict.
  • test_includes_core_scalar_fields — Snapshot includes marking id, code,
    inscription, state, town, captured_at.
  • test_includes_citation_rows — Snapshot lists citations with expected detail.

ComputePayloadDiffTest
  • test_diff_lists_changed_fields_only — Diff only changed keys; excludes
    captured_at from meaningful diff.
  • test_diff_handles_added_and_removed_keys — Added/removed keys appear with
    null before/after as appropriate.
  • test_diff_handles_non_dict_inputs — Non-dict inputs yield empty diff.

LogSubmissionTransactionTest
  • test_records_action_and_actor — Logged transaction has actor, marking,
    action; diff_payload includes changed field (e.g. code).
  • test_anonymous_actor_is_stored_as_null — Anonymous-like actor yields null
    actor on transaction.

CreateMarkingVersionTest
  • test_version_numbers_increment_per_marking — Three versions on same marking
    get version_no 1, 2, 3.
  • test_version_is_marking_scoped — Second marking’s first version is 1 even
    if first marking already has versions.

RestoreMarkingFromSnapshotTest
  • test_restores_scalar_fields — After manual field changes, restore from
    snapshot puts code and inscription back.
  • test_restore_recreates_citations — Restore removes extra citations and
    leaves snapshot’s citation set.
  • test_restore_short_circuits_for_invalid_snapshot — Invalid snapshot type does
    not change marking.

MarkingListFilterTest
  • test_type_filter_matches_exact_type — type=TOWNMARK returns only townmarks
    by code set equality.
  • test_state_filter_matches_name_or_abbrev — state=Virginia vs state=VA same
    results.

  • test_shape_filter_by_id — Filter by shape id excludes manuscript rows

  • test_is_manuscript_true — Only manuscript marking when true.
  • test_is_manuscript_false — Excludes manuscript-only row when false.
  • test_is_manuscript_ignored_for_other_values — Non true/false value ignored

  • test_earliest_use_year_min_filter — Min year filter excludes markings whose
    earliest seen is before threshold.
